# Artifact Center — 设计规范（DESIGN）

> **本文档是全项目视觉与交互的最高准则。**  
> 所有页面、组件、动效、AI 生成 UI，必须符合本文。  
> 冲突时：UI 原则 < 组件指南 < **本 DESIGN** < 产品愿景。

---

## 1. 设计使命

让企业内网制品管理拥有 **Premium SaaS** 体验：冷静、克制、快速、可信。

用户打开系统应感觉：

> 「这是一个认真做产品的团队做的工具。」

而不是：

> 「又一个 Vue Admin / Ant Design Pro 套壳。」

---

## 2. 设计关键词（必须体现）

| 关键词 | 落地含义 |
|--------|----------|
| Minimal | 去掉一切不帮助决策的元素 |
| Modern | 2020s 产品语言：清晰层级、无拟物堆砌 |
| Professional | 企业可信，不玩具化 |
| Premium | 材质、间距、字体像 Linear / Vercel |
| Elegant | 少即是多，对齐与节奏统一 |
| Large Spacing | 呼吸感，拒绝信息堵塞 |
| Large Typography | 标题有存在感，正文可读 |
| Card Based | 对象用卡片承载，而非密表 |
| Fast | 感知性能：即时反馈、骨架克制 |
| Simple | 一条主路径走完任务 |

---

## 3. 绝对禁止（Zero Tolerance）

以下任一出现即视为设计不合格：

| 禁止项 | 说明 |
|--------|------|
| 中国式 ERP 风格 | 深蓝顶栏 + 密菜单 + 灰底表单墙 |
| 信息过密 | 行高过小、边距 < 规范、一屏塞满 |
| 满屏 Table | 默认视图不是 Excel；表格仅在「密集数据任务」可选 |
| Dashboard 首页 | 禁止饼图/折线/六色统计卡墙作为首页 |
| 五颜六色 | 禁止多色标签狂欢；状态色 ≤ 语义必需 |
| 重阴影 | 禁止大面积 `shadow-2xl` 漂浮卡片 |
| 花哨动画 | 禁止弹跳、闪烁、无意义 parallax |
| Bootstrap 风格 | 圆角按钮彩虹、默认蓝链接堆砌 |
| Ant Design Pro 默认皮肤 | 侧栏深色 + 顶栏 + 面包屑 + ProTable 套路照搬 |

**补充禁止：**

- 渐变按钮到处用
- 每个卡片不同彩色左边条
- 3D 图标、拟物文件夹
- 大红大绿成功/失败铺满屏幕
- 弹窗里再开多层抽屉无节制
- 无意义插画占满首屏

---

## 4. 体验标杆（对标方式）

对标的是 **气质与交互质量**，不是抄功能：

| 产品 | 学习点 |
|------|--------|
| Linear | 密度可控、键盘、状态色克制、列表优雅 |
| GitHub | 对象页信息架构、元数据可读、操作分区 |
| Vercel | 空状态、状态徽章、极简顶栏 |
| Raycast | 搜索为中心、结果清晰 |
| Apple HIG | 字体层级、间距节奏、动效服务于理解 |

**内化一句：**  
界面应像「精心排版的工具」，而不是「后台管理系统模板」。

## 5. 设计系统基础

### 5.1 色彩

采用 **中性色主导 + 单一品牌强调色**。

#### 语义结构

| Token | 用途 |
|-------|------|
| `background` | 页面底 |
| `surface` | 卡片 / 面板 |
| `surface-muted` | 次级区域、输入底 |
| `border` | 分割与描边 |
| `border-strong` | 强调描边（focus 外） |
| `text-primary` | 主文案 |
| `text-secondary` | 次文案 |
| `text-tertiary` | 辅助 / 时间戳 |
| `brand` | 主操作、关键链接、焦点 |
| `brand-foreground` | 品牌色上的文字 |
| `success` / `warning` / `danger` / `info` | 仅状态 |

#### 使用规则

1. **80%+ 界面由中性色构成**
2. Brand 色用于：主按钮、关键导航指示、焦点环、关键数据高亮
3. 状态色不得用于装饰
4. 暗色模式：降低纯白对比，避免 `#000` 大面积（可用深灰表面）
5. 图表若出现（非首页核心）：单色或双色序列，禁止彩虹

#### 建议方向（实现时映射到 CSS 变量）

- 浅色：接近冷灰背景（非纯蓝灰 ERP）
- 深色：接近 Zinc/Neutral，而非纯黑 + 霓虹
- Brand：一种沉稳色（如墨蓝、靛、或极轻的中性黑按钮体系——二选一要统一）

**推荐默认策略：**  
主按钮可用 **近黑/深石墨**（浅色模式）营造 Premium；Brand 色用于链接与 focus。  
或主按钮用 Brand，但全局只保留 **一个** 强调色相。

### 5.2 字体

| 层级 | 建议 | 用途 |
|------|------|------|
| Display | 28–36px / semibold | 极少：落地/空状态标题 |
| H1 | 24–28px / semibold | 页面标题 |
| H2 | 18–20px / semibold | 区块标题 |
| H3 | 15–16px / medium | 卡片标题 |
| Body | 14–15px / regular | 正文 |
| Small | 12–13px / regular | 辅助信息 |
| Mono | 12–13px | 哈希、版本号、文件名 |

**规则：**

- 页面标题必须有存在感（Large Typography）
- 正文不低于 14px（桌面）
- 字重克制：常规 + medium + semibold，避免满屏 bold
- 数字与哈希使用等宽，提升专业感
- 中英混排注意断行与标点

**字体选择建议：**

- 西文：Inter / Geist / system-ui 堆栈
- 中文：系统栈优先（PingFang / Microsoft YaHei UI）或统一产品字体
- 不要用「粗圆体 / 艺术字体」

### 5.3 间距系统

基于 **4px** 网格，常用：

`4, 8, 12, 16, 20, 24, 32, 40, 48, 64`

| 场景 | 建议 |
|------|------|
| 页面左右边距 | ≥ 24px；宽屏内容区 max-width + 居中 |
| 页面顶边到标题 | 24–40px |
| 卡片内边距 | 16–24px |
| 卡片间距 | 12–16px |
| 表单项间距 | 16–20px |
| 列表行高（紧凑） | 仍保持可点区域 ≥ 40px |

**Large Spacing 原则：**  
宁可一屏少几条，也不要挤。  
筛选条与内容之间要有明确呼吸。

### 5.4 圆角

| 元素 | 圆角 |
|------|------|
| 按钮 / 输入框 | 8–10px |
| 卡片 | 12–16px |
| 弹层 / Dialog | 16px |
| 徽章 | 999px（pill）或 6px |
| 头像 | 全圆或 8px 方圆统一 |

全局圆角档位不超过 3 种，避免有的 2px 有的 24px。

### 5.5 边框与阴影

**边框优先于阴影。**

- 默认卡片：`1px solid border`，无阴影或极轻阴影
- 悬停：边框略强化或背景微变，**不要** 突然重阴影浮起
- 下拉/弹层：可用中等阴影（一层），仍保持低对比
- 禁止多层彩色阴影

### 5.6 图标

- 仅用 **Lucide**（线性图标）
- 默认 16 / 20px，stroke 与产品一致
- 不与文字抢层级：图标用 secondary 色，除非可点
- 禁止每个菜单项不同彩色填充图标

## 6. 布局结构

### 6.1 总体骨架

```
┌──────────────────────────────────────────────────────────┐
│ Topbar：Logo | 全局搜索 | 快捷上传 | 用户               │
├────────────┬─────────────────────────────────────────────┤
│            │  Page Header（标题 + 主操作）                │
│  Sidebar   │  Toolbar（筛选 / 视图切换）                  │
│  (可选窄)  │  Content（卡片流 / 列表）                    │
│            │                                             │
└────────────┴─────────────────────────────────────────────┘
```

**说明：**

- Sidebar 保持窄、项少（≤ 7 一级项）
- 可用 **无侧栏 + 顶栏导航** 的更轻方案（项目级页面）
- 内容区有最大宽度（如 1200–1440px）避免超宽拉伸难读
- 详情页可用 **主栏 + 右栏元数据**（GitHub 风格）

### 6.2 页面类型模板

#### A. 索引页（项目列表 / 制品列表）

1. 大标题 + 一句描述（可选）+ Primary CTA  
2. 筛选工具条（少而精）  
3. 内容：卡片网格或优雅列表  
4. 空状态居中、可操作  

#### B. 详情页

1. 返回 + 标题 + 状态徽章  
2. 主操作右上（下载、分享、更多）  
3. 主信息区（文件、版本、备注）  
4. 侧栏：哈希、上传者、时间、项目、构建信息  

#### C. 创建/上传流

1. 专注模式：减少旁支导航干扰  
2. 步骤尽量 1–2 屏完成  
3. 大拖拽区 + 清晰进度  
4. 成功后给出「查看详情 / 继续上传 / 复制链接」  

#### D. 设置页

- 左侧小节导航 + 右侧表单  
- 危险操作分区并二次确认  

### 6.3 栅格

- 项目卡片：自适应网格，`minmax(280px, 1fr)`
- 制品：默认列表（可读性）；提供「舒适卡片」视图切换
- 间距统一，最后一行不对齐时保持左对齐

## 7. 组件视觉语言

### 7.1 按钮

| 类型 | 样式 | 用途 |
|------|------|------|
| Primary | 实心（深色或 brand） | 每区最多 1 个主操作 |
| Secondary | 浅底或描边 | 次要操作 |
| Ghost | 透明 | 工具栏、行内 |
| Danger | 危险色实心/描边 | 删除等 |

规则：

- 同一视图 **唯一主按钮**
- 禁用态降低透明度且不可点，并说明原因（tooltip 或辅助文案）
- 加载态用按钮内 spinner，保持宽度防抖动

### 7.2 输入

- 高度舒适（36–40px）
- 聚焦：清晰 focus ring（brand 低透明），不要粗黑框
- 错误：边框 danger + 下方文案，不弹系统 alert
- 标签在上，不模拟老式左右错位表单

### 7.3 卡片

- 承载 **对象**（项目、制品）
- 结构：标题行 → 关键元数据 → 次要信息 → 悬停操作
- 整卡可点进入详情；破坏性操作收入菜单
- 选中/拖拽态边框提示

### 7.4 列表

当数据需要扫描对比时可用列表，但必须：

- 行距舒适
- 列 ≤ 6 个关键列，其余进详情
- 表头轻量
- 行悬停微背景
- 支持列少而精，而不是「字段全展示」

### 7.5 徽章（Badge）

用于：类型（APK/AAB/EXE/ZIP）、环境、状态。

- 中性底 + 深字 优先
- 状态才用淡色底（success/warn/danger 的 **浅** 变体）
- 同一行徽章 ≤ 3 个

### 7.6 导航

- 当前项：明确但不刺眼（左侧细指示条或轻底）
- 不要「选中高亮大彩块」

### 7.7 弹层

- Dialog：确认、短表单
- Sheet/Drawer：中等复杂度（移动端友好）
- Command Palette：搜索与跳转（Raycast 感）
- 禁止层层叠叠超过两层用户可理解深度

### 7.8 空状态

- 图标轻量 + 标题 + 一句话 + 主操作
- 文案专业、具体（「还没有制品」+「上传第一个包」）
- 不要尴尬大插画淹没布局

### 7.9 加载

- 优先骨架屏（与最终布局同构）
- 局部 spinner 用于按钮/刷新
- 禁止全屏闪白无结构

### 7.10 Toast / 反馈

- 成功：轻 toast，自动消失
- 错误：可读原因 + 可重试
- 不打断主流程的，不用模态

## 8. 关键页面设计规格

### 8.1 登录页

- 居中卡片或左右分栏（右品牌极简）
- 无多余营销轮播
- 品牌名 Artifact Center + 简短副标题
- 表单字段最少

### 8.2 项目列表

- 标题「Projects」
- 搜索项目 + New Project
- 卡片：项目名、描述摘要、最近更新、制品数量（可选一个数字即可）
- **不要** 项目统计仪表盘

### 8.3 项目主页

- 最近制品
- 快捷上传入口
- 筛选入口到完整列表
- 可放「Activity」时间线（克制），不是图表墙

### 8.4 制品列表

工具条建议：

`类型` `环境` `状态` `时间` `上传者` | `排序` | `视图切换` | `上传`

内容行/卡展示最少集：

- 文件名 / 展示名
- 类型徽章
- 版本
- 渠道/环境
- 时间
- 上传者

悬停：下载、复制链接、更多

### 8.5 上传页

1. 选择项目（若上下文未带）  
2. 拖拽文件区（大、清晰、支持多文件 P1）  
3. 识别类型后展示元数据表单  
4. 上传进度：确定、可取消  
5. 完成态：清晰 CTA  

进度条用 brand/中性，不要彩虹条。

### 8.6 制品详情

**视觉重点：可信与可操作。**

- 文件大号标题 + 类型图标
- 主按钮：Download
- 次按钮：Copy link / Share
- 元数据定义列表（Definition list）优雅排版
- 哈希可一键复制
- 备注 Markdown 可选（P1）
- 危险操作在 More 菜单

### 8.7 搜索

- 顶栏搜索唤起面板或跳转结果页
- 结果分组：Projects / Artifacts
- 键盘上下选择，Enter 打开
- 高亮匹配片段（克制）

## 9. 动效（Framer Motion）

### 9.1 原则

**动效只解释状态变化，不表演。**

| 允许 | 禁止 |
|------|------|
| 150–250ms 的 fade / slight-move | 弹跳、旋转进场 |
| 列表错落 20–40ms stagger（克制） | 长时序编排动画 |
| 布局微调 layout 动画 | 滚动视差花活 |
| 进度指示 | 成功彩带、烟花 |

### 9.2 推荐参数

- ease：`ease-out` 或标准 product easing
- 距离：位移 ≤ 8–12px
- 尊重 `prefers-reduced-motion`

### 9.3 页面切换

- 同布局内内容淡入即可
- 不要整站翻页特效

## 10. 内容设计（UX Writing）

### 10.1 语气

- 专业、简洁、肯定
- 中文产品语气，可保留关键名词英文（Artifact、Project、SHA-256）
- 不卖萌、不官腔、不恐吓

### 10.2 示例

| 场景 | 推荐 | 避免 |
|------|------|------|
| 空列表 | 还没有制品。上传第一个构建包吧。 | 暂无数据！！ |
| 上传成功 | 已上传。链接已就绪。 | 恭喜您操作成功 |
| 无权限 | 你没有下载权限。请联系项目管理员。 | 非法访问 |
| 删除确认 | 将归档「app-1.2.3.apk」。列表中默认隐藏。 | 请慎重！！！ |

### 10.3 用词统一

| 统一用 | 不用 |
|--------|------|
| 制品 / Artifact | 附件、网盘文件 |
| 项目 / Project | 应用空间（除非产品改名） |
| 上传 | 投递、录入 |
| 废弃 / 归档 | 作废拉黑 |
| 下载 | 拉取（对用户文案） |

## 11. 响应式与密度

### 11.1 断点策略

- 桌面优先（内网工作站为主）
- 平板：侧栏可折叠
- 手机：详情与下载必须可用（测试现场扫码场景 P1）

### 11.2 密度

默认 **Comfortable**。  
高级用户可提供 **Compact** 列表密度（P1），但仍禁止 ERP 密表。

---

## 12. 深色模式

- 设计系统从第一天用 CSS 变量，便于 dark
- 深色不是「反色」，是单独调过的 surface 层级
- 边框在深色中更重要
- 状态色降低饱和，避免霓虹

MVP 可先做好浅色，变量结构必须预留 dark。

---

## 13. 无障碍（A11y）

- 对比度：正文达到 WCAG AA
- 焦点可见，禁止 `outline: none` 无替代
- 交互元素有可访问名称
- 仅靠颜色区分状态时，需搭配图标或文案
- 弹层焦点陷阱与 ESC 关闭

---

## 14. 设计质量检查清单

每个页面发版前自检：

- [ ] 是否像 Linear/Vercel 多于像后台模板？
- [ ] 是否存在无意义颜色与重阴影？
- [ ] 标题与留白是否足够「Premium」？
- [ ] 主操作是否唯一且明确？
- [ ] 列表是否只展示决策字段？
- [ ] 空/加载/错误态是否完整？
- [ ] 动效是否可关掉且无花活？
- [ ] 文案是否专业统一？
- [ ] 是否误做了 Dashboard 首页？
- [ ] 是否出现满屏 Table 作为默认视图？

任一项失败 → 重做该 UI，而不是「先上线再改」。

## 15. 与工程落地的映射

| 设计决策 | 工程 |
|----------|------|
| Token 化颜色/圆角/阴影 | Tailwind + CSS variables |
| 组件 | shadcn/ui 定制，不接受默认「后台感」皮肤裸用 |
| 图标 | lucide-react |
| 动效 | framer-motion，按本规范限幅 |
| 字体 | 全局 typography 插件或 CSS |

**禁止** 为了快直接粘贴 Ant Design Pro 布局骨架。

---

## 16. 设计决策记录（ADR 摘要）

| 决策 | 选择 | 原因 |
|------|------|------|
| 列表形态 | 卡片 + 优雅列表双视图 | 对象感与扫描效率平衡 |
| 首页 | 项目/工作入口 | 反对 Dashboard 形式主义 |
| 强调色 | 单一 brand + 中性主按钮策略 | Premium、少彩 |
| 阴影 | 边框优先 | 现代、干净 |
| 导航 | 浅侧栏或顶栏，项少 | 降低认知负担 |

新增重大视觉决策请追加本节，勿静默偏离。

---

## 17. 结语

Artifact Center 的设计竞争力不是「功能菜单多少」，而是：

**在内网环境里，仍然坚持克制、优雅与速度。**

所有 AI 与工程师：先读本文，再写 UI。
