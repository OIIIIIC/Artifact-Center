# Artifact Center — Design System（设计系统）

> **文档定位**  
> 本文是 Artifact Center 的 **可执行设计系统规范**。  
> 它不是情绪板，不是灵感合集，而是约束未来所有页面、组件、AI 生成 UI 的硬规则。
>
> **文档关系**  
> - `00-VISION`：产品为何存在  
> - `02-DESIGN`：产品级视觉与交互准则  
> - `07-UI-PRINCIPLES`：决策否决权  
> - **`08-DESIGN-SYSTEM`（本文）**：Token、组件行为、布局节奏、检查清单  
>
> **冲突裁决**  
> 愿景 > DESIGN > 本文 > 组件指南 > 个人偏好。  
> 任何实现若违反本文的 Never 条款，视为不合格交付，不得「先上线再美化」。
>
> **工程映射（未来）**  
> 所有视觉值必须落到 CSS 变量 / Tailwind token，禁止页面内魔法数散落。  
> 本文 **不写 HEX 色值**；色相与亮度在主题实现阶段按本文语义锁定。

# 1. Design Philosophy（设计哲学）

## 1.1 一句话目标

把企业内网的制品管理做成 **冷静、克制、快速、可信的专业软件**——  
用户感觉像在用 Linear / GitHub，而不是又一个后台模板。

核心用户任务只有一条链路：

**找到正确的包 → 确认可信 → 下载或分享。**

设计系统的一切决策，都服务这条链路。

## 1.2 设计目标（必须同时成立）

| 原则 | 含义 | 为什么这样设计 | 何时体现 | 何时禁止误用 |
|------|------|----------------|----------|--------------|
| **Less but Better** | 少做装饰，把决策字段做对 | 内网工具失败点是噪音，不是功能不够 | 每个控件、每行文案都要过「删掉会怎样」 | 禁止用「多几个入口显得功能全」堆菜单 |
| **Content First** | 内容与对象优先于框架 | 用户来看制品，不是来看导航皮肤 | 页面 70%+ 面积给对象列表/详情 | 禁止厚重双栏装饰、大插画抢主内容 |
| **Card Based** | 项目/制品以卡片或优雅列表呈现 | 实体感帮助扫描与决策 | 索引页、摘要入口 | 禁止把卡片当万能容器塞表单墙 |
| **Large Typography** | 标题有存在感，正文可读 | Premium 感来自层级，不是渐变 | 页面 H1、空状态、详情文件名 | 禁止为「高级」把正文做成 12px 密排 |
| **Large Spacing** | 统一呼吸感 | 密度是气质，ERP 靠挤显得「信息多」 | 页面边距、卡片内边距、区块间距 | 禁止为塞更多行而压缩到规范以下 |
| **Calm UI** | 安静、中性、少彩 | 状态色才有意义；彩虹无语义 | 默认中性表面 + 单一强调 | 禁止装饰性色块、彩色左边条列表 |
| **Fast Interaction** | 反馈即时、路径短 | 感知速度 = 产品信任 | Hover 100ms 内、骨架同构、搜索快 | 禁止用长动画伪装性能 |
| **Professional Software** | 企业可信、可审计、文案克制 | 制品涉及发布与责任 | 哈希、上传者、时间、权限提示清晰 | 禁止卖萌文案、恐吓式报错、玩具化图标 |

## 1.3 Never（绝对禁止）

以下任一出现，**整页否决**，不接受局部修补了事：

| Never | 为什么禁止 | 常见伪装（同样禁止） |
|-------|------------|----------------------|
| **Dashboard 首页** | 首页应是工作入口，不是 KPI 表演 | 六色统计卡 + 折线/饼图墙、「运营总览」 |
| **满屏 Table 作为默认视图** | Excel 墙摧毁对象感与扫描节奏 | Ant Design ProTable 默认密度照搬 |
| **五颜六色** | 颜色失去语义，界面廉价 | 每行不同标签色、彩虹进度条 |
| **重阴影** | 制造虚假「高级感」，破坏平面秩序 | `shadow-2xl` 卡片瀑布、多层投影 |
| **复杂渐变** | 与专业工具气质冲突 | 渐变按钮、渐变顶栏、渐变卡片底 |
| **过度动画** | 拖慢感知、干扰决策 | Bounce、Elastic、视差、进场编排秀 |
| **ERP 风格** | 信息密度竞赛，不是产品 | 深蓝顶栏 + 密菜单 + 灰表单墙 |
| **OA 风格** | 流程表格化、文牍化 | 红头文件感、审批流皮肤默认化 |
| **Bootstrap 风格** | 2000s 组件默认美学 | 粗边框默认蓝、圆角按钮彩虹 |
| **Ant Design Pro 默认后台皮肤** | 全网同质化内网后台 | 深色侧栏 + 顶栏 + 面包屑 + ProTable 套路 |

**补充 Never：**

- 每个菜单项不同彩色填充图标  
- 卡片左侧彩色竖条作为「设计语言」  
- 成功撒花、全屏大红大绿提示  
- 弹窗套抽屉套弹窗（深度 > 2 必须改信息架构）  
- 「暂无数据！！」「操作成功！！」类廉价文案  

## 1.4 Always（默认成立）

1. 一页一个主任务，一区一个 Primary 按钮  
2. 中性色 ≥ 界面 80%  
3. 边框分区优先于阴影  
4. 搜索是一级能力（顶栏或 Command Palette）  
5. 空 / 加载 / 错误 / 无权限 四态完整  
6. 关掉动效后功能完整

# 2. Color System（色彩系统）

## 2.1 总原则

**为什么：**  
颜色是最贵的注意力。制品平台需要用户信任元数据，不需要视觉派对。  
**中性色主导 + 单一强调色 + 语义状态色**，才能让状态在安静界面中「跳出来」。

**怎么用：**  
所有颜色以 **语义 Token** 命名，不在业务代码写具体色值。  
实现阶段再绑定 Light / Dark 两套值；本文只规定 **角色与用法**。

## 2.2 Token 地图

### Primary（品牌 / 强调）

| 项 | 规定 |
|----|------|
| **用途** | 焦点环、关键链接、导航当前指示、需要品牌记忆的高亮；可选用于 Primary 按钮（见按钮章） |
| **为什么** | 全站只允许 **一个** 强调色相，建立识别且避免「后台皮肤五彩」 |
| **何时使用** | 用户需要注意到「可行动 / 当前所在」时 |
| **禁止** | 大面积铺底、每个图标染色、成功/失败也用 Primary、多 Primary 色相并存 |

**主按钮策略（二选一，全站锁死一种）：**

- **策略 A（推荐 Premium）：** Primary 按钮 = 近黑 / 深石墨；Brand 色用于链接与 focus  
- **策略 B：** Primary 按钮 = Brand 色；则链接与按钮同色相，需严格控制用量  

选定后写入主题实现，禁止页面级混用。

### Neutral（中性）

覆盖：`background`、`surface`、`surface-muted`、`text-primary`、`text-secondary`、`text-tertiary`、`border`、`border-strong`。

| 项 | 规定 |
|----|------|
| **用途** | 构成 80%+ 界面：页面底、卡片、正文、分割 |
| **为什么** | Calm UI 的骨架；对比度层级代替「用颜色分区」 |
| **何时使用** | 默认一切；不确定时选中性 |
| **禁止** | 用高饱和灰蓝当 ERP 底；正文与背景对比不足；三级文字过浅导致不可读 |

**文本层级规则：**

- `text-primary`：标题、关键值、可点主链接  
- `text-secondary`：说明、次要元数据  
- `text-tertiary`：时间戳、辅助提示  
- 禁止三级文字做可点主操作（可点至少 secondary + 明确 affordance）

### Success / Warning / Danger / Info

| Token | 用途 | 何时使用 | 禁止 |
|-------|------|----------|------|
| **Success** | 成功、可用、active 的肯定语义 | Toast 成功、状态徽章「正常/已完成」 | 装饰、大面积成功底铺屏 |
| **Warning** | 需注意但未失败 | deprecated、即将过期、校验警告 | 普通提示滥用警告色 |
| **Danger** | 不可逆/破坏性 | 删除、永久清除、危险确认 | 普通 Secondary 操作、仅「重要」就标红 |
| **Info** | 中性说明性提示 | 系统说明、帮助提示条 | 代替 Primary 吸引点击 |

状态色必须提供 **浅底 + 深字** 或 **图标 + 文案** 组合，禁止只靠色点区分。

### Border

| 项 | 规定 |
|----|------|
| **用途** | 卡片轮廓、输入框、分割线、表格线（若使用） |
| **为什么** | 本系统用边框建立结构，替代重阴影 |
| **何时** | 默认卡片 1px `border`；悬停可切到 `border-strong` 或微提背景 |
| **禁止** | 2px+ 粗黑框当默认；彩色边框当装饰；用边框画满密宫格 |

### Background / Surface

| Token | 用途 | 规则 |
|-------|------|------|
| **Background** | 页面画布 | 应略区别于 Surface，形成「纸叠在底上」的一层关系 |
| **Surface** | 卡片、下拉、对话框主体 | 高于 Background 的一层；同层 Surface 不对彼此叠重阴影 |
| **Surface-muted** | 输入底、代码块底、次级分组 | 不用于大面积主内容区 |

**禁止：** Background 与 Surface 完全同色导致卡片「糊」在页面上却又不用边框补偿。

## 2.3 语义使用配额

| 规则 | 说明 |
|------|------|
| 中性占比 | ≥ 80% 像素为中性 |
| 强调色 | 单屏可感知强调点 ≤ 3 处（含主按钮） |
| 状态色 | 仅徽章、Toast、行内状态、错误边框 |
| 类型色 | APK/AAB/EXE/ZIP 用 **固定低饱和映射** 或全中性徽章；禁止高饱和四色对撞 |

## 2.4 Dark Mode（色彩层要求）

详见第 15 章。色彩系统层硬性要求：

- Dark 不是 Light 反相  
- 禁止纯黑大底 + 纯白字的极端对比作为默认  
- 状态色在 Dark 下 **降饱和**，禁止霓虹  
- Border 在 Dark 中更重要（分隔靠线，不靠阴影）

# 3. Typography（字体系统）

## 3.1 为什么建立完整字阶

没有字阶就没有 Premium。  
ERP 界面的典型失败是：**所有字差不多大、差不多重**，只能靠颜色和表格线分区。  
本系统用 **尺寸 + 字重 + 颜色层级** 三者分工，而不是只用 bold。

## 3.2 字体族

| 角色 | 选择 | 为什么 | 禁止 |
|------|------|--------|------|
| 界面正文 | Inter / Geist / system-ui 堆栈 | 现代产品默认可读性 | 艺术字体、粗圆体、衬线装饰体做 UI |
| 中文 | 系统优先（如 PingFang SC / Microsoft YaHei UI） | 内网部署可控、渲染稳定 | 为「好看」引入未授权商用字体文件而不评估授权 |
| 等宽 | 系统 mono 或 Geist Mono / UI Mono | 哈希、版本、文件名对齐专业感 | 正文大段用等宽 |

## 3.3 字阶表（桌面默认）

数值为实现目标区间；落地时映射为 token，全站不得另起炉灶。

| Token | 字号 | 字重 | 行高 | 使用场景 | 禁止 |
|-------|------|------|------|----------|------|
| **Display** | 28–36px | Semibold (600) | 1.2–1.25 | 极少：营销向落地（若有）、超大空状态标题 | 普通业务页标题；一屏多个 Display |
| **H1** | 24–28px | Semibold | 1.25–1.3 | 页面标题（Projects、Artifacts、详情文件名可等同） | 卡片标题；侧栏项 |
| **H2** | 18–20px | Semibold | 1.3 | 区块标题、Dialog 标题 | 表格列头 |
| **H3** | 15–16px | Medium (500) | 1.35 | 卡片主标题、列表主字段 | 替代 Body 大段正文 |
| **Title** | 14–15px | Medium | 1.4 | 紧凑列表主标题、菜单项强调 | 与 H3 混用无规范；同页随意切换 |
| **Body** | 14–15px | Regular (400) | 1.5 | 正文、表单值、说明段落 | < 14px 桌面正文；大段 400 字堆在卡片里 |
| **Caption** | 12–13px | Regular | 1.4 | 时间、次要元数据、脚注 | 主操作文案；关键错误仅 Caption 且对比不足 |
| **Label** | 12–13px | Medium | 1.3 | 表单标签、字段名 | 用占位符代替 Label |
| **Button** | 13–14px | Medium | 1 | 按钮文字（单行） | 按钮内两行字；按钮用 Light 字重发虚 |
| **Code** | 12–13px | Regular mono | 1.4 | SHA、versionCode、packageName、路径 | 长文说明用 Code 装酷 |

## 3.4 使用规则

**何时放大标题：**  
每个一级页面必须有清晰 H1。这是 Large Typography 的最低交付线。

**何时用弱化文字：**  
L3 信息（时间、上传者）用 Caption + `text-secondary/tertiary`。  
禁止用缩小字号却保持 `text-primary` 造成「又小又抢」。

**字重纪律：**

- 默认 Regular + Medium + Semibold 三档  
- 禁止满屏 Bold / Black  
- 禁止用字重代替颜色表达错误（错误仍需文案 + 边框）

**中英混排：**

- 产品名、Artifact、SHA-256 可保留英文  
- 中英文之间按可读性加空格（实现与文案规范统一）  
- 避免标题中英文各用一套冲突字体导致基线跳动过大

## 3.5 禁止事项（字体）

- 同一组件内超过 3 种字号  
- 用斜体表达「提示」（中文 UI 斜体观感差）  
- 全大写英文长句（降低可读性）  
- 为对齐英文设计而牺牲中文行高

# 4. Spacing System（间距系统）

## 4.1 为什么用 4px 网格

统一间距 = 统一节奏。  
没有网格时，每个页面「差一点」的 margin 会累积成廉价感。  
4 的倍数便于 Tailwind / 设计工具对齐，也便于 Code Review 一眼否决魔法数。

## 4.2 间距阶梯

| Token | 值 | 典型用途 |
|-------|-----|----------|
| `space-1` | **4** | 图标与紧邻文字间隙、Badge 内微垫 |
| `space-2` | **8** | 紧凑组内间距、按钮 icon gap |
| `space-3` | **12** | 卡片间距（紧）、列表项内元素 |
| `space-4` | **16** | 卡片内边距下限、表单字段间常规 |
| `space-5` | **20** | 表单字段舒适间距（可选） |
| `space-6` | **24** | 页面水平 padding 下限、卡片内边距推荐、区块小分隔 |
| `space-8` | **32** | 页面标题区与内容区间距、Section 间距 |
| `space-10` | **40** | 宽屏页头上下呼吸 |
| `space-12` | **48** | 大区块分隔、空状态内部节奏 |
| `space-16` | **64** | 极少：着陆分区、极端留白 |

## 4.3 场景处方

| 场景 | 使用 | 为什么 | 禁止 |
|------|------|--------|------|
| 页面左右 padding | ≥ 24；宽屏配合 max-width | 内容不贴边 | 16 以下桌面页边 |
| 页头到内容 | 24–40 | 建立页面层级 | 标题贴筛选条 |
| 卡片 padding | 16–24 | 对象可读 | 8px 卡片内边（变 ERP 密盒） |
| 卡片与卡片 | 12–16 | 网格可扫 | 0 间距粘连；>24 导致网格散架无结构 |
| 表单项之间 | 16–20 | 可点可填 | 字段间距 4–8 挤成墙 |
| 筛选条与列表 | ≥ 16 | 工具与内容分离 | 筛选贴第一张卡 |
| 图标按钮热区 | 视觉 16–20 icon，热区 ≥ 32–40 | 可点 | 热区 = 图标像素 |

## 4.4 Large Spacing 执行定义

**允许一屏少显示几条，也不允许行高/边距跌破上表下限。**  
Compact 模式（P1 可选）只能在 **列表行** 收紧 1 档，不得同时压缩页边距与卡片 padding。

## 4.5 禁止

- 随意 5、7、9、11、13 等非网格值（除非 1px 边框类）  
- 用负 margin 硬对齐（优先改结构）  
- 「看起来空就再塞一个字段」——空是功能，不是缺陷

# 5. Radius System（圆角系统）

## 5.1 为什么限制档位

圆角是材质语言。  
档位过多 = 像拼凑组件库。  
全站 **≤ 4 个半径档 + 1 个 pill**。

## 5.2 档位

| Token | 值 | 用于 | 不用于 |
|-------|-----|------|--------|
| `radius-sm` | **6–8** | Badge（非 pill）、小控件、Checkbox 外观 | 大卡片 |
| `radius-md` | **8–10** | **Button、Input、Select、小 Menu 项容器** | Dialog |
| `radius-lg` | **12–16** | **Card、下拉面板** | 按钮（避免按钮过圆） |
| `radius-xl` | **16–20** | **Dialog / Sheet 容器** | 行内小元素 |
| `radius-full` | **9999** | Pill Badge、Avatar 圆 | 大卡片、主按钮（主按钮保持 md） |

推荐默认锁定：

- Button / Input：**8**  
- Card：**12 或 14**  
- Dialog：**16**  
- 不设 **24** 作为默认业务圆角（过大显消费级 App，可仅用于特殊营销容器，业务后台默认禁用）

## 5.3 使用与禁止

| 何时使用更大圆角 | 何时禁止 |
|------------------|----------|
| 浮层需要与页面「分层」时用 xl | 同一卡片内外圆角乱跳（外 16 内 2） |
| Badge 状态可用 pill 增强可扫性 | 所有容器都 pill 导致无法对齐网格 |
| Avatar 统一圆或统一方圆 | 有的 2px 有的 24px 无 token |

# 6. Shadow System（阴影系统）

## 6.1 哲学

**Border + Surface 分层，是本产品的默认结构语言。**  
阴影是 **浮层特权**，不是卡片默认皮肤。

**为什么：**  
重阴影是「后台模板高级感」的廉价配方，会与 Calm UI 冲突，并在暗色模式脏污。

## 6.2 阴影档位（语义，非具体值）

| Token | 强度 | 用途 |
|-------|------|------|
| `shadow-none` | 无 | 默认卡片、列表行、页面模块 |
| `shadow-xs` | 极轻（可选） | 需要与底极微分层但仍保持平面时 |
| `shadow-sm` | 轻 | Dropdown、Popover、Tooltip |
| `shadow-md` | 中 | Dialog、Command Palette |

禁止定义 `shadow-xl` / 彩色阴影作为产品 token。

## 6.3 何时允许 Shadow

1. **浮层**：Dialog、Dropdown、Popover、Command Palette、Toast（可选极轻）  
2. **临时抬升**：拖拽中的对象（交互态，结束后恢复）  
3. **不可用边框表达的重叠**：浮层盖住复杂背景时  

## 6.4 何时禁止 Shadow

1. 默认 Card 网格靠阴影漂浮  
2. Hover 时突然从 none → 重阴影「浮起」  
3. 每个统计块一层阴影堆立体感  
4. 文字阴影、发光阴影、Brand 色阴影  
5. 为了「高级」给侧栏/顶栏加厚投影  

## 6.5 Hover 正确做法

优先：

- 背景 `surface` → 微提亮/微压暗  
- 或 `border` → `border-strong`  
- 光标与可点暗示  

而不是：

- 位移 + 重阴影 + 缩放 0.98 的夸张组合

# 7. Grid（栅格与页面节奏）

## 7.1 为什么要最大内容宽

超宽屏把一行拉到 2000px 会降低扫读效率。  
专业工具把内容约束在 **可读 transcribe 宽度**，两侧留呼吸。

## 7.2 布局 Token

| Token | 规定 | 为什么 |
|-------|------|--------|
| **App max content width** | 主内容 **1200–1440px** 居中（实现锁一个值，如 1280） | 列表与详情可读 |
| **Reading width（可选）** | 长文说明 **680–720px** | 设置说明、帮助文案 |
| **Page horizontal padding** | ≥ 24；大屏 32 | 防贴边 |
| **Page top padding** | 24–40 | 页头存在感 |
| **Section gap** | 32–48 | 区块分离 |
| **Card gap** | 12–16 | 网格节奏 |
| **Sidebar width** | 220–260（若有） | 窄、稳、项少 |
| **Detail two-column** | 主栏 ~2/3 + 侧栏 ~1/3；侧栏 min ~280 | GitHub 式元数据栏 |

## 7.3 栅格行为

### 项目卡片网格

- `minmax(280px, 1fr)` 自适应  
- 列数随宽度变化，**左对齐**，不强制拉伸最后一张变形  

### 制品列表

- 默认：单列优雅列表（非满屏多列表格）  
- 可选：卡片视图切换  
- 工具条全宽，内容区遵守 max-width  

### 何时打破 max-width

- 全宽上传拖拽区在专注流中可接近内容宽上限  
- 禁止审计表为「显得厉害」无边距通栏到屏幕边缘  

## 7.4 页面垂直节奏（标准索引页）

```
[ Topbar ]
[ Page padding-top ]
[ PageHeader: H1 + 描述 + 主操作 ]
[ gap 16–24 ]
[ Toolbar: 筛选 / 视图 ]
[ gap 16 ]
[ Content: 卡片或列表 ]
[ gap 32 ]
[ Pagination / 加载更多 ]
```

禁止：顶栏下直接表格顶满；禁止 Header 与表格之间无 Toolbar 却塞三行统计。

# 8. Card Design（卡片设计 · 重点）

## 8.1 卡片是什么

卡片 = **一个可理解业务对象的摘要容器**。  
在 Artifact Center 中，主要对象是 **Project** 与 **Artifact**。

**为什么 Card Based：**  
对象有边界，用户才能「点进某个东西」，而不是在表格单元格里游泳。

## 8.2 卡片必须包含的结构

```
┌─────────────────────────────────────────┐
│  Title（H3/Title）           Status?    │
│  Subtitle / key meta（版本、类型…）      │
│                                         │
│  Secondary meta（时间、上传者…）         │
│                                         │  ← 留白：内容不贴边、区块有呼吸
│  (Hover: 快捷操作显现)                  │
└─────────────────────────────────────────┘
```

| 要素 | 要求 | 为什么 |
|------|------|--------|
| **留白** | padding 16–24；内部块间距 ≥ 8 | 可读、可点 |
| **标题** | 单行优先，溢出省略；是最强文字 | 扫读锚点 |
| **副标题 / 关键元数据** | 版本、类型、环境中决策字段 | 不进详情也能判断「是不是这个包」 |
| **状态** | Badge，最多与类型合计 ≤ 3 个徽章 | 状态可见但不噪 |
| **Hover** | 背景或边框微变；快捷操作出现 | 暗示可交互 |
| **点击反馈** | 整卡可进详情；active/focus 可见 | 对象导航 |

## 8.3 交互规则

| 规则 | 说明 |
|------|------|
| 整卡可点 | 默认导航到详情；使用语义化链接 |
| 内部按钮 | 下载、更多等：停止冒泡；键盘可到达 |
| 悬停操作 | 默认 1 主（如下载）+ 1 更多；**总数 ≤ 3** |
| 焦点 | 键盘 focus 时显示环，不得 `outline: none` 无替代 |
| 选中（若多选） | 边框/轻底，不用重阴影 |

## 8.4 信息配额（硬限制）

单卡 **默认可见字段 ≤ 6**（含标题）。  
更多进详情。

推荐 Artifact 卡字段集：

1. 名称  
2. 类型  
3. 版本  
4. 环境或渠道（择一主显示）  
5. 时间  
6. 上传者  

## 8.5 禁止

| 禁止 | 原因 |
|------|------|
| 卡片内塞完整表单 | 卡片不是设置页 |
| 卡片内 ≥ 4 个实心按钮 | 破坏主操作唯一性 |
| 彩色左边条 + 多色图标 | ERP/模板味 |
| 阴影漂浮 + 渐变底 | 违反 Calm |
| 卡片内嵌迷你图表 | Dashboard 化 |
| 无标题只有一堆 Badge | 无法扫读 |
| 不同页面卡片圆角/padding 私自改 | 系统崩溃 |

## 8.6 卡片 vs 列表行

| 用卡片 | 用优雅列表行 |
|--------|----------------|
| 项目入口、强调实体浏览 | 制品需要快速纵向扫描 |
| 信息较少、视觉入口强 | 字段需对齐比较（仍限列） |

列表行仍遵守 Large Spacing 与悬停规则，**不是退回满屏 Table**。

# 9. Button System（按钮系统）

## 9.1 变体

| 变体 | 视觉 | 用途 | 何时用 | 禁止 |
|------|------|------|--------|------|
| **Primary** | 实心（近黑或 Brand） | 该区域最重要行动 | 上传、下载、创建、保存 | 一页/一区多个 |
| **Secondary** | 描边或浅底 | 次要行动 | 取消、次要分流 | 伪装成第三个 Primary |
| **Ghost** | 透明底 | 工具栏、轻操作 | 筛选旁、行内次要 | 主 CTA |
| **Text / Link** | 无容器文字按钮 | 内联导航、次要链接 | 文案流中 | 破坏性操作 |
| **Icon Button** | 方热区图标 | 更多、关闭、复制 | 有 `aria-label` | 无障碍名称缺失 |
| **Danger** | 危险色实心或描边 | 破坏性确认 | 删除/永久清除确认 | 归档可逆操作用 Danger 吓人 |

## 9.2 尺寸

仅三档：`sm` / `md` / `lg`。

| 尺寸 | 高度目标 | 场景 |
|------|----------|------|
| sm | ~32 | 表格行内、密集工具（仍保持热区） |
| md | 36–40 | **默认** |
| lg | 40–44 | 页头主操作、空状态 CTA |

## 9.3 Primary 数量铁律

| 范围 | Primary 上限 |
|------|----------------|
| 一个 **页面可见主区域** | **1** |
| 一个 **Dialog** | **1**（Danger 确认可替代 Primary，二者不并存为两个实心主按钮） |
| 一个 **卡片 Hover 条** | **0–1** 实心，其余 Ghost |

**为什么：**  
多个 Primary = 没有重点 = 降低决策速度。

## 9.4 Danger 规则

**何时可用：**

- 不可逆删除、清空、移除成员且后果严重  
- 必须配合确认 Dialog，文案含对象名  

**何时禁止：**

- 退出登录  
- 取消  
- 归档/废弃（优先 Warning 语义或中性确认，除非产品定义归档=高危）  
- 仅因为「管理员功能」就标红  

## 9.5 状态

| 状态 | 要求 |
|------|------|
| Default / Hover / Active | 100ms 内可感知 |
| Disabled | 降透明 + 不可点；尽量说明原因（tooltip） |
| Loading | 按钮内 spinner，**宽度不抖动**，防重复提交 |

## 9.6 禁止

- 渐变按钮、闪光按钮  
- 按钮内两行文字  
- 图标五颜六色  
- Primary 用 Success 绿「看起来积极」

# 10. Input（输入系统）

## 10.1 通用规则

| 规则 | 为什么 |
|------|--------|
| Label 在上且始终可见 | 占位符会消失，无障碍与扫读差 |
| 高度 36–40（md） | 与按钮系统对齐，可点 |
| Focus ring 清晰（Brand/中性高对比） | 键盘用户生命线 |
| 错误：边框 Danger + 下方文案 | 禁止 alert 弹窗报错 |
| Hint 用 Caption + tertiary | 不与错误文案抢 |

## 10.2 各控件

### Input（单行）

| 何时用 | 禁止 |
|--------|------|
| 名称、版本号、渠道等短文本 | 用 Input 塞长备注（改 Textarea） |
| 可前缀/后缀（单位、图标）但保持克制 | 每框彩色左边条 |

### Textarea

| 何时用 | 禁止 |
|--------|------|
| 备注、描述 | 高度拖成半屏却无 max-length 提示（长文本场景要说明） |
| 最小 3 行视觉高度 | 与 Input 同高伪装 |

### Search

| 规则 | 说明 |
|------|------|
| 位置 | 顶栏一级；结果页可有大搜索框 |
| 行为 | 防抖；Esc 清空/关闭；Enter 打开首个结果（在 Palette 中） |
| 视觉 | 中性底，**不要**做成厚重彩条搜索 |
| 禁止 | 搜索框五颜六色；无结果时空白无说明 |

### Upload

| 规则 | 说明 |
|------|------|
| 形态 | 大 Dropzone + 清晰边界（虚线边框可用，但保持中性） |
| 状态 | 空 / 拖入悬停 / 上传中 / 成功 / 失败 |
| 进度 | 确定进度条；中性或单一强调色 |
| 禁止 | 彩虹进度；上传区插满装饰插画；失败只 spinner 转 |

详见第 13 章进度。

### Dropdown / Select

| 规则 | 说明 |
|------|------|
| 长列表 | 可搜索（项目、类型） |
| 浮层 | shadow-sm + border，项高舒适 ≥ 36 |
| 禁止 | 原生 select 在关键流无样式放任到破版；无限嵌套子菜单 |

## 10.3 表单布局

- 单列优先（创建/上传元数据）  
- 相关字段分组，组间距 ≥ 24  
- 字段间距 16–20  
- 主提交在表单末端或页头，**不两边各一个保存**  

## 10.4 禁止

- 左右 label 密表格式（ERP）默认化  
- 红星乱飞却无分组  
- 提交后才报 12 个错且不定位字段

# 11. Navigation（导航系统）

## 11.1 目标

用户始终知道：

1. 我在哪  
2. 如何返回  
3. 如何搜索  
4. 本页主操作是什么  

## 11.2 Sidebar

| 项 | 规定 |
|----|------|
| **是否必须** | 可选；也可用顶栏主导航换更轻壳 |
| **宽度** | 220–260 |
| **一级项数量** | **≤ 7** |
| **当前项** | 轻底或细指示，禁止大彩块 |
| **图标** | Lucide 单色，与文字二级色 |

**何时用侧栏：** 全局模块稳定（Projects / Artifacts / Settings）。  
**禁止：** 15 项一级菜单、双层厚侧栏、深色霓虹侧栏皮肤。

## 11.3 Topbar

必须：

- 产品标识  
- **全局搜索入口**  
- 用户菜单  
- 可选：快捷上传  

禁止：

- 顶栏堆 10 个彩按钮  
- 顶栏做公告跑马灯  

## 11.4 Breadcrumb

| 何时用 | 何时不用 |
|--------|----------|
| 深度 ≥ 3 且层级真实（项目 → 制品） | 只有一级页面硬造面包屑 |
| 项少、可点击祖先 | 又长又臭、含不可点无意义节点 |

视觉：Caption/Body 弱化，当前项 primary。

## 11.5 Command Palette / Search

| 项 | 规定 |
|----|------|
| **地位** | 一级能力（Raycast 感） |
| **触发** | 顶栏点击 + 键盘快捷键（实现时定义） |
| **结果** | 分组：Projects / Artifacts；键盘上下 + Enter |
| **禁止** | 搜索只藏在侧栏深处；结果无分组无空态 |

## 11.6 首页导航原则

首页 / 默认登录落地：

- **项目列表** 或 **继续上次工作的入口**  
- 可有「最近制品」摘要  
- **不是 Dashboard**  

**禁止首页出现：**

- 统计卡墙  
- 多图表面板  
- 「待办审批中心」OA 布局  
- 为管理者做的运营驾驶舱（用量放 Settings，不放首页）

# 12. Empty State（空状态）

## 12.1 为什么空状态是产品

空状态是用户第一次「失败」的瞬间——没有数据。  
处理好，用户知道下一步；处理差，用户觉得系统坏了。

## 12.2 结构（固定四段）

```
[ 可选：轻量图标 / 极简插画 ]
[ 标题：发生了什么 ]
[ 描述：一句话原因或指引 ]
[ CTA：一个主按钮 + 可选次链接 ]
```

| 元素 | 规范 |
|------|------|
| **插画** | 可选；线性、单色或双色极简；**不占半屏**；无插画可用 Lucide 大图标（muted） |
| **标题** | H2/H3，具体：「还没有制品」 |
| **描述** | Body，一行优先：「上传第一个构建包，或向项目管理员申请权限。」 |
| **CTA** | **一个** Primary（如上传）；次要用 Text |

## 12.3 场景文案方向

| 场景 | 标题方向 | CTA |
|------|----------|-----|
| 无项目 | 还没有项目 | 创建项目 |
| 无制品 | 还没有制品 | 上传 |
| 搜索无结果 | 没有匹配的结果 | 清空筛选 |
| 无权限 | 无法查看 | 联系管理员（非假按钮） |

## 12.4 禁止

- 「暂无数据」「null」「——」无解释  
- 搞笑表情包、过度插画  
- 空状态同时 3 个 Primary  
- 用 Error 红色渲染「只是空」

# 13. Loading（加载系统）

## 13.1 原则

**骨架优先，Spinner 克制。**  
Spinner 表达「局部在忙」；骨架表达「结构已知，内容将至」。

## 13.2 Skeleton

| 规则 | 说明 |
|------|------|
| 同构 | 骨架形状 ≈ 最终列表/卡片布局 |
| 数量 | 列表 3–8 行足够，不要 20 行闪烁 |
| 颜色 | 中性脉冲，低对比 |
| 禁止 | 全屏白闪无结构；骨架与真布局错位导致跳动 |

## 13.3 Spinner

| 何时用 | 何时不用 |
|--------|----------|
| 按钮提交中 | 整页首次加载（用骨架） |
| 刷新图标位 | 每个卡片一个转圈 |
| 极短等待的局部 | 遮罩整站无法取消又无文案 |

## 13.4 Progress（确定进度）

| 场景 | 要求 |
|------|------|
| 上传 | 百分比或字节进度；可取消（若技术允许） |
| 批处理 | 汇总进度 + 当前文件名 |
| 颜色 | 单一中性/强调，禁止彩虹条 |
| 完成 | 进入成功态或列表，不靠撒花 |

## 13.5 Upload 加载专项

1. 选择后立即出现队列项  
2. 上传中：进度 + 文件名 + 取消  
3. 失败：错误原因 + 重试  
4. 成功：短反馈 + 查看详情 / 继续上传  

禁止：上传中只能看全屏 spinner，无法感知文件是否在动。

## 13.6 禁止

- 每个 API 都全页 Loading 遮罩  
- 骨架结束与数据到达之间布局狂跳（保留最小高度）  
- 用假进度条（不确定就用不确定态，但要诚实）

# 14. Motion（动效系统）

## 14.1 哲学

动效只做三件事：

1. 解释状态变化  
2. 连接空间关系（谁从哪出现）  
3. 提供瞬时反馈  

不做：表演、品牌秀、拖延。

必须支持 `prefers-reduced-motion: reduce` → 基本淡入或无动画。

## 14.2 时长表

| 类型 | 时长 | Easing | 用途 |
|------|------|--------|------|
| **Hover / Active 反馈** | **100ms** | ease-out | 颜色、边框、背景 |
| **微移 / Fade** | **120–150ms** | ease-out | Tooltip、小面板 |
| **Page content 切换** | **150ms** | ease-out | 同壳内内容淡入 |
| **Modal / Dialog** | **180ms** | ease-out | 出现/消失；位移 ≤ 8–12px |
| **Dropdown** | **120–160ms** | ease-out | 轻 fade + 微移 |
| **复杂序列** | 默认 **禁止** | — | — |

位移默认 **≤ 8–12px**。透明度变化优先。

## 14.3 允许

- 列表极轻 stagger（项间 20–40ms，总时长可控）  
- Layout 微调（折叠筛选）  
- 进度条宽度过渡  

## 14.4 禁止

| 禁止 | 原因 |
|------|------|
| Bounce | 玩具感 |
| Elastic / 弹簧过度 | 拖沓 |
| 旋转翻页、立方体切换 | 干扰 |
| Parallax 滚动 | 无信息增益 |
| 成功彩带/粒子 | 与专业软件冲突 |
| > 300ms 的常规 UI 过渡 | 感觉慢 |
| 无限脉冲吸引注意（除骨架） | 焦虑 |

## 14.5 验收三问（每个动画）

1. 关掉后功能完整吗？  
2. 是否让界面感觉更慢？  
3. 是否帮助理解状态？  

不通过则删除。

# 15. Dark Mode（暗色模式）

## 15.1 为什么不是反相

反相会制造：

- 刺眼纯白字  
- 边框消失  
- 状态色变成霓虹  
- 阴影脏斑  

暗色是 **另一套 Surface 层级设计**。

## 15.2 设计规则

| 规则 | 说明 |
|------|------|
| Surface 层级 | background < surface < elevated surface，靠亮度阶梯 + border |
| 文本 | 降低纯白；正文用高但不刺眼的中性亮色 |
| 边框 | 比浅色模式更依赖 border 分隔 |
| 阴影 | 更少、更弱；浮层靠边框 + 轻微 shadow |
| 状态色 | 降饱和；浅底徽章重新调，不直接反相 Light token |
| 图片/图标 | Lucide 随 currentColor；避免假设黑图标 |

## 15.3 实现要求

- 从第一天 CSS 变量分主题，禁止组件内写死浅色值  
- MVP 可先交付 Light，但 **不得** 用无法扩展的硬编码阻碍 Dark  
- 主题切换不得打乱间距与字阶（只换颜色 token）

## 15.4 禁止

- `#000` 大底 + `#fff` 正文当默认  
- Dark 下重阴影卡片  
- Brand 色发光描边  
- 仅反转 hue 交差  

## 15.5 验收

- 关键页面（列表、详情、上传、Dialog、Palette）在 Dark 下对比度达标  
- 状态徽章可读  
- Focus ring 可见

# 16. Icon System（图标系统）

## 16.1 唯一来源

**仅使用 Lucide Icons。**

**为什么：**  
线宽一致、现代、与 Linear/Vercel 气质接近；混用多套图标是后台拼盘的标志。

## 16.2 规格

| 项 | 规定 |
|----|------|
| 默认尺寸 | **16**（行内）、**20**（导航/强调） |
| 特殊 | 空状态可用 28–32，仍保持线型 |
| 颜色 | 默认 `text-secondary`；可点/激活随语义 |
| 描边 | 跟随 Lucide 默认，不私自改成填充插画风 |
| 对齐 | 与文字光学对齐，gap 8 |

## 16.3 使用场景

| 场景 | 规则 |
|------|------|
| 导航 | 图标 + 文字；不靠彩色图标辨识 |
| 类型 | APK/AAB 等可用文件/包相关图标，**单色** |
| 装饰 | 尽量不用纯装饰图标 |

## 16.4 禁止

- Font Awesome + Material + Ant 图标混用  
- 每个菜单不同填充彩图标  
- 3D / 拟物 / 品牌表情包当系统图标  
- 图标按钮无文案且无 `aria-label`

# 17. Accessibility（无障碍）

## 17.1 为什么要做

内网专业软件仍需键盘、对比度与读屏基本可达。  
无障碍不是加分项，是 **可聚焦、可完成任务** 的底线。

## 17.2 键盘

| 要求 | 说明 |
|------|------|
| 全关键路径可达 | 搜索、上传、下载、确认 Dialog |
| Tab 顺序合理 | 不跳进隐藏层 |
| Esc | 关闭 Dialog / Palette / 下拉 |
| Enter | 激活主按钮/选中项 |
| 快捷键 | 搜索类全局快捷键不与输入冲突时启用 |

## 17.3 Focus

| 要求 | 说明 |
|------|------|
| 可见焦点环 | 禁止无替代地去掉 outline |
| 焦点环样式 | 与 Brand/高对比 token 一致，全站统一 |
| 焦点恢复 | Dialog 关闭后回到触发器 |

## 17.4 颜色

| 要求 | 说明 |
|------|------|
| 对比度 | 正文与背景达到 WCAG AA |
| 非唯一信道 | 状态不只靠红绿；配合文案/图标 |
| 色盲 | 类型区分优先形状/文字，不只靠色相 |

## 17.5 可点击区域

| 要求 | 说明 |
|------|------|
| 最小热区 | 建议 ≥ 32×32，舒适 40×40 |
| 列表行 | 高度保证可点 |
| 间距 | 相邻危险操作不贴死 |

## 17.6 其他

- 表单错误与控件 `aria-invalid` / 描述关联  
- 图片/装饰图标不干扰读屏  
- 动效可关  

## 17.7 禁止

- 仅用颜色标错  
- 点击热区小于图标像素  
- 自动播放无控动画

# 18. Responsive（响应式）

产品以 **桌面工作站** 为第一场景，但测试现场存在移动端下载需求。

## 18.1 断点策略（语义）

| 档位 | 典型宽度 | 布局策略 |
|------|----------|----------|
| **Desktop** | ≥ 1280 | 完整壳：侧栏可展开、双栏详情、舒适网格 |
| **Laptop** | 1024–1279 | 完整功能；侧栏可窄或可折叠；保持 max-width 节奏 |
| **Tablet** | 768–1023 | 侧栏收为抽屉；列表优先；详情单栏，元数据下置 |
| **Mobile** | < 768 | 顶栏精简；搜索可进二级；主 CTA 可达；下载路径最短 |

实现时用统一 breakpoint token，避免页面私定。

## 18.2 Desktop

| 要求 | 禁止 |
|------|------|
| Large Spacing 全开 | 为「显示更多」压成密表 |
| 双栏详情 | 无意义三栏 |
| Hover 快捷操作可用 | 仅靠 Hover 才能完成 **唯一** 主路径（键盘/触控需替代） |

## 18.3 Laptop

| 要求 | 说明 |
|------|------|
| 与 Desktop 同信息架构 | 减少列数，不删功能 |
| 网格列数减少 | 保持卡片 padding |

## 18.4 Tablet

| 要求 | 说明 |
|------|------|
| 导航折叠 | 不复制一套完全不同的 IA |
| 筛选 | 可进 Sheet；默认露出最关键 1–2 个筛选 |
| 禁止 | 把桌面表直接横向挤压出微小列 |

## 18.5 Mobile

| 要求 | 说明 |
|------|------|
| 详情下载一等公民 | 大按钮 Download |
| 元数据折叠 | 先看版本/类型/时间 |
| 上传 | 可用但允许简化；P1 可强调扫码下载页 |
| 禁止 | 桌面 Hover 菜单无触控入口；字号 < 14 正文硬塞 |

## 18.6 触控与 Hover 共存

- Hover 增强桌面  
- 触控设备：关键操作常显或进明确「更多」  
- 禁止仅悬浮才出现的 **唯一下载按钮**

# 19. UI Checklist（交付检查清单）

每个页面 / PR / AI 生成 UI，合并前必须自检。  
**任一项失败 → 不合并。**

## 19.1 结构与焦点

- [ ] 页面只有 **一个** 主要任务  
- [ ] 页面只有 **一个** 视觉重点（H1 / 主对象）  
- [ ] 可见主区域 **Primary Button ≤ 1**  
- [ ] 无 Dashboard 统计卡墙 / 图表墙  
- [ ] 无 ERP / OA / Bootstrap / Ant Design Pro 默认皮肤气质  

## 19.2 视觉系统

- [ ] 字号层级正确（H1 / 正文 / Caption 可区分）  
- [ ] 中性色主导，无五颜六色装饰  
- [ ] 间距落在 4px 网格，卡片留白 ≥ 16  
- [ ] 圆角仅使用系统档位  
- [ ] 默认卡片无重阴影，结构靠 Border + Surface  
- [ ] Hover 反馈与全站一致（非重阴影起飞）  

## 19.3 组件与状态

- [ ] Card 字段未超载，按钮未堆砌  
- [ ] 空状态含标题 + 描述 + CTA（如适用）  
- [ ] Loading 用骨架或正确进度，无 Spinner 滥用  
- [ ] 错误态可读且可恢复  
- [ ] 无权限态明确，非空白  

## 19.4 动效与主题

- [ ] 动效时长符合 100 / 150 / 180 体系  
- [ ] 无 Bounce / Elastic / 花哨序列  
- [ ] `prefers-reduced-motion` 可降级  
- [ ] 颜色全部走 Token，具备 Dark 扩展能力  
- [ ] Dark 模式关键页可读（若已启用）  

## 19.5 无障碍与响应式

- [ ] Focus 可见  
- [ ] 图标按钮有 accessible name  
- [ ] 可点击区域足够  
- [ ] 状态不只靠颜色  
- [ ] Laptop / Tablet / Mobile 无破版、主路径可达  

## 19.6 内容

- [ ] 文案专业、具体、非公文腔  
- [ ] 危险操作有对象名确认  
- [ ] 信任信息（哈希/上传者/时间）在详情路径可见

# 20. Visual Benchmark（视觉对标）

## 20.1 只允许参考的对象

| 对象 | 学习什么 | 不学习什么 |
|------|----------|------------|
| **Apple Human Interface** | 留白、字体层级、克制动效、控件清晰 | 消费级拟物、营销落地页套路硬搬到工具 |
| **Linear** | 密度可控、键盘、状态色克制、列表优雅 | 复制其品牌色与像素级组件 |
| **GitHub** | 对象页 IA、元数据分区、操作克制 | 照搬其全部信息密度与历史包袱 |
| **Vercel** | 极简顶栏、状态徽章、空状态气质 | 部署产品特有概念硬套 |
| **Raycast** | 搜索即入口、结果清晰、快 | 桌面启动器交互全盘搬进 Web 壳 |
| **Notion** | 内容优先、块面安静、页面呼吸 | 文档编辑器复杂嵌套当制品列表 |

## 20.2 对标方式（强制）

学习的是：

- 信息层级  
- 留白节奏  
- 交互克制  
- 专业冷静的气质  

**不是：**

- 直接复制 UI 像素  
- 复制 Logo 附近装饰  
- 复制对方品牌色当本产品 Primary  
- 截图贴组件当设计稿而不经本系统 Token 转化  

## 20.3 禁止对标

以下来源 **不得** 作为视觉参考主来源：

- 国内传统 ERP / OA 后台  
- 任意 Admin 模板市场默认皮肤  
- Bootstrap 演示站  
- Ant Design Pro 官网默认示例布局（可作为工程组件参考，**不可作为视觉目标**）  
- 炫技 Dribbble 大仪表盘  

## 20.4 评审句式

评审时只允许类似判断：

- 「这个详情页的元数据栏接近 GitHub 的清晰度，通过。」  
- 「这个首页是 Dashboard，否决。」  
- 「间距像 Linear，颜色像模板站，重做颜色。」  

不允许：

- 「先模仿 XX 后台，用户习惯。」——与愿景冲突，直接否决。
