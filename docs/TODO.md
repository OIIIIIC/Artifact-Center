## 后端就绪后 — 优先

### 操作记录（审计日志）

**优先级：P1（后端可用后第一批产品能力）**

**一句话：**  
记录「谁在什么时间对什么对象做了什么」，用于内网追责与协作排查。

**建议记录的操作：**

| 类型 | 示例                                              |
| ---- | ------------------------------------------------- |
| 应用 | 创建、编辑、改生命周期、归档、删除                |
| 制品 | 上传/发布、标记最新、改渠道、废弃                 |
| 分发 | **下载**、**通过分享链接下载**、生成/撤销分享链接 |
| 权限 | 成员角色变更、重置密码（若保留）                  |
| 设置 | 保留策略变更等                                    |

**建议字段：**

- 时间、操作者（用户 id / 姓名）
- 动作类型（枚举）
- 对象类型 + 对象 id（应用 / 制品 / 分享链接）
- 摘要文案（给人看的一行）
- 可选：IP、User-Agent、结果成功/失败、变更 diff

**产品入口（建议，后端后做）：**

1. **应用详情 →「活动」或「操作记录」Tab**（按该应用过滤）
2. **全局设置 → 审计**（管理员看全站，可筛选导出）
3. 分享链接落地页 **不** 对访客展示全量审计，仅服务端记日志

**注意：**

- 与「应用设置 / 全局设置」区分：操作记录是只读时间线，不是配置页
- 下载类高频操作需限流/采样策略（后端设计时考虑）
- 日志应不可被普通用户篡改（见 `03-ARCHITECTURE`）

**依赖：** 真实用户身份、API、持久化存储。
