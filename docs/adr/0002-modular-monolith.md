# ADR-0002: Modular Monolith 架构

- **状态**: accepted
- **日期**: 2026-07-19
- **决策者**: 产品规范 (03-ARCHITECTURE.md)

## 背景

Artifact Center 是企业内网部署的制品管理平台，面向中小团队。需要平衡开发效率、运维成本和架构灵活性。

## 决策

采用 **Modular Monolith** 架构：API、领域服务、Worker 同进程/同仓库，逻辑上按模块分界。

模块划分：

| 模块     | 职责                       |
| -------- | -------------------------- |
| identity | 用户、会话、OIDC/LDAP 适配 |
| project  | 项目、成员、角色           |
| artifact | 制品元数据、状态机         |
| storage  | 预签名上传/下载、分片      |
| search   | 检索索引                   |
| audit    | 操作与下载日志             |
| parser   | APK 等异步解析             |

## 替代方案

- **微服务**: 运维成本过高，与内网小团队部署场景不匹配。规范明确"不引入与问题规模不匹配的微服务爆炸"。
- **纯单体无模块边界**: 长期维护困难，不利于后续拆分。

## 后果

- **正面**: 部署简单(Docker Compose 一键启动)，开发迭代快，运维成本低
- **负面**: 模块边界可能随代码增长模糊，需持续 discipline 维护
- **演进**: 预留模块化边界，若某模块性能或组织需要可独立拆分
