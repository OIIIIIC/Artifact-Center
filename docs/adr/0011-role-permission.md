# ADR-0011: 项目级角色权限模型

- **状态**: accepted
- **日期**: 2026-07-19
- **决策者**: 产品规范 (01-PRD.md §6, 10-DATABASE.md §3.1)

## 背景

Artifact Center 需要权限控制来保证制品安全。不同角色对应用有不同的操作权限。MVP 阶段不需要复杂的 RBAC/ABAC 模型。

## 决策

采用**项目级(应用级)角色模型**，三级权限：

| 角色       | 查看 | 下载 | 上传 | 编辑元数据 | 废弃/归档 | 删除 | 管理设置 |
| ---------- | ---- | ---- | ---- | ---------- | --------- | ---- | -------- |
| Viewer     | ✓    | ✓    | -    | -          | -         | -    | -        |
| Maintainer | ✓    | ✓    | ✓    | ✓          | ✓         | -    | -        |
| Admin      | ✓    | ✓    | ✓    | ✓          | ✓         | ✓    | ✓        |

系统级 **Super Admin** (users.role = 'admin') 管理全局配置与用户，可访问所有应用。

实现细节：

- `users.role` 存平台级角色
- `application_members` 存应用级关系，唯一键 `(application_id, user_id)`
- 创建应用时创建者自动成为 Maintainer
- 后端在应用、制品、分享、搜索和审计查询时执行应用级授权

## 替代方案

- **纯平台级角色**: 粒度太粗，无法做到"不同应用不同权限"
- **文件级 ACL**: MVP 过度设计，PRD 明确"先项目级角色，再文件级(如需)"
- **RBAC 角色继承**: 当前规模不需要

## 后果

- **正面**: 简单直观，满足 MVP 所有权限场景
- **负面**: 如需更细粒度(如"仅可下载特定渠道制品")需升级模型
- **演进**: 保留扩展空间，成员管理接口已就绪，前端界面待完善
