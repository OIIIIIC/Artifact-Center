# ADR-0012: 下载链接安全策略 (预签名 URL)

- **状态**: proposed
- **日期**: 2026-07-19
- **决策者**: 产品规范 (03-ARCHITECTURE.md §4.3, §7.2)

## 背景

制品下载是核心功能，但需要防止未授权访问和链接泄露。MVP 当前使用 multipart 上传和受控流式下载。长期需要更安全、更高性能的下载方案。

## 决策

- MVP: 鉴权后受控流式输出下载
- 推荐演进: **预签名 URL** 方案
  - 客户端请求下载 → 后端鉴权 + 记审计 → 生成时效预签名 URL → 重定向
  - 预签名 URL 有时效限制，过期自动失效
- **禁止公网裸奔永久链接**，除非有时效签名
- 分享链接走 `/public/shares/:token`，不再提供按制品 ID 公开下载的接口

## 替代方案

- **永久公开链接**: 严重安全风险，明确禁止(架构原则: "安全默认关闭公网匿名下载")
- **仅受控流式输出**: 当前 MVP 方案，简单但大文件时 API 进程内存压力大
- **Token-based 下载(非预签名)**: 仍需经过 API 进程，未解决大文件流式传输性能问题

## 后果

- **正面**: 预签名 URL 将下载流量从 API 进程卸载到对象存储/反向代理，大幅提升大文件下载性能
- **负面**: 需要接入 MinIO/S3 或配置 Nginx internal redirect；MVP 阶段暂不实现
- **状态**: proposed — 待 P2 Usability 或接入对象存储时正式 accepted
